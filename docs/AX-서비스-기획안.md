# AI 네이티브 AX 전환 웹 서비스 — 기획안 v0.1

> 리서치 산출물 (재무·인사·사업기획·영업·마케팅 5개 팀 도메인 분석 + 기술 아키텍처)

## 0. 제품 비전

계산은 **결정론적 코드가 100% 책임**지고, 이상징후·개선점 도출은 **Claude AI가 근거를 인용한 리포트로 보조**하며, 최종 결론과 의사결정은 **사람**이 내리는 **3단 구조**의 AI 네이티브 AX 전환 웹 서비스.

> 사람이 하던 일을 AI가 대체하는 것이 아니라 — 단순계산은 코드로 덜어주고(계산 부담 제거), AI는 "왜 그런지·무엇을 개선해야 하는지"를 검증 가능한 근거와 함께 리포트하며(판단 보조), 사람은 숫자 만들기가 아닌 판단·승인·전략에 집중한다.

```
[원본 데이터] → (코드) 계산·집계·정합성검증·이상치 플래그 → CRO
                                                          ↓
                              (AI) 이상징후 해석·개선점·우선순위 리포트(근거 인용)
                                                          ↓
                                  (사람) 리포트 검토 → 승인/반려 → 의사결정
```

## 1. 설계 원칙

1. **계산-해석 분리 (Calc=Code, Meaning=AI)**: 모든 수치는 결정론적 엔진의 CRO에서만 나오고, AI는 CRO 밖의 숫자를 절대 생성하지 않는다.
2. **결론은 항상 사람 (Human-in-the-loop)**: 모든 리포트는 Draft로 생성되고, 사람 승인 전에는 다운스트림에 노출되지 않는다.
3. **근거 인용 강제 (Evidence-first)**: 리포트의 모든 finding은 유효한 evidence_ref(CRO 항목ID/RAG chunk)를 동반. 불일치 시 자동 재생성 후 사람 검토 큐.
4. **Garbage-in 차단**: ValidationEngine이 치명 오류 발견 시 AI 호출 게이트를 닫아 그럴듯한 거짓 리포트를 원천 봉쇄.
5. **공통 추상화 재사용**: 도메인 무관 코어 + 도메인 플러그인으로 팀 확장 시 동일 골격 재사용.
6. **보안·권한을 1급 시민으로**: tenantId/role 전 계층 전파, 필드 레벨 RBAC·마스킹·AI 전송 전 비식별화.
7. **완전 추적성 (Audit by design)**: 입력 해시·CRO버전·모델/프롬프트버전·승인 이력을 불변 로그로 기록.

## 3. 공통 기능 (Cross-cutting)

- **DataConnector**: ERP·회계SW·HRIS·CRM·광고API·엑셀/CSV를 단일 `ingest()`로 추상화. 소스별 어댑터는 플러그인. (MVP는 엑셀/CSV)
- **CalculationEngine + ValidationEngine**: 도메인 무관 결정론적 계산 코어 + 도메인 플러그인. 모든 산술·정합성·규칙 이상치 → CRO. 치명 오류 시 AI 게이트 차단.
- **ReportEngine (리포트 허브)**: CRO+검증+RAG근거 → Claude로 구조화 리포트. 프롬프트 캐싱·근거검증 후처리·재생성 루프·Draft 상태관리.
- **알림·조기경보**: 임계값/통계 플래그 + AI 해석 리스크를 웹/메일/메신저로. 일정 기반 + 이상징후 기반 통합.
- **협업·Human-in-the-loop**: Draft→검토→승인/반려/코멘트→재생성. 팀 간 데이터 공유(재무↔기획↔영업↔마케팅).
- **EvidenceLedger / AuditLog**: 입력해시·CRO버전·모델/프롬프트버전·승인이력을 traceId로 불변 기록.
- **TenantContext + 보안·RBAC**: tenantId/role 전 계층 전파, Postgres RLS·필드 마스킹·AI 전송 전 비식별화.

| 데이터 레이어 | 소스 수집·원본 스냅샷·CRO/리포트/감사로그/RAG 영속화 | Postgres+pgvector, 오브젝트 스토리지, KMS/Secrets |

### Claude API 활용
- **기본 모델**: `claude-opus-4-8` (다단계 인과 추론·우선순위화에 적합, 1M 컨텍스트, adaptive thinking). 대량·정형 분류 작업은 `claude-haiku-4-5`/`claude-sonnet-4-6`로 라우팅.
- **구조화 출력**: `output_config.format` (json_schema) 또는 LangChain `withStructuredOutput`(Zod). evidence_refs는 CRO 항목ID만 허용 → 환각 차단. (4.8은 prefill 미지원)
- **Prompt caching**: 안정적 시스템 프롬프트·공통 회계기준 문서를 prefix에 두고 `cache_control` 배치. 가변값(기간/회사ID/계산결과)은 breakpoint 뒤.
- **비용·지연**: 야간 대량 리포트 Message Batches(50% 할인), 인터랙티브 Q&A 스트리밍, effort/adaptive thinking 조절.

### LangChain.js 패턴
- "얇게, 오케스트레이션 글루로만" — 핵심 추론은 결정론적 엔진에, LangChain은 체인 구성·구조화 파싱·RAG·관찰가능성에만.
- **Chain 우선(LCEL), Agent는 제한적**: 리포트 파이프라인은 고정 단계 → 워크플로 체인(감사·재현 유리). tool-calling agent는 인터랙티브 Q&A 한정(읽기전용 화이트리스트 도구).
- **RAG는 근거 보강용**(수치 산출 불가): 회계정책·인사규정·KPI정의·업종 벤치마크·과거 리포트를 테넌트 격리 벡터스토어에.

## 5. MVP 추천 — 재무팀 Finance AX

**추천 범위**: '자금일보·현금흐름 예측 + 유동성 경보' + '월 결산 자동화 + 이상 분개 탐지' 수직 슬라이스 (엑셀/CSV 단일 소스)

**근거**:
- **임팩트**: 재무는 simple-calc 비중이 가장 높고(매일/매월 반복 고난도 계산), 유동성 위기·이상 분개 리스크가 커 AI 보조 가치가 명확.
- **실현가능성**: 재무 지표는 공식·단위·기간이 명확한 결정론적 계산 → 100% 자동화 쉬움. '코드→AI→사람' 루프와 환각 차단을 가장 깔끔하게 실증.
- **데이터 접근성**: 엑셀/CSV 업로드로 ERP 연동 없이 즉시 PoC. 재무의 CRO·검증 패턴이 이후 팀 확장의 레퍼런스.

**성공 지표**: (1) 리포트 산출 리드타임 80%+ 단축, (2) CRO 일치율 100%(환각 0건), (3) AI 리포트 사람 채택률 70%+, (4) 베타 재무팀 주 1회 이상 정착.

| **0 — MVP** | 0~3개월 | 재무·엑셀 단일 슬라이스로 '코드→AI→사람' 3단 루프 + 환각 차단 실증 |
| **1 — 멀티테넌시·보안** | 3~6개월 | TenantContext·RLS·RBAC·시크릿·ERP 커넥터·prompt caching·Batches |
| **2 — 도메인 확장** | 6~9개월 | 인사·영업·마케팅·기획을 공통 추상화 플러그인으로, PII 비식별화 |
| **3 — RAG 근거 보강** | 9~12개월 | 규정·벤치마크·과거 리포트 인덱싱, 해석 단계 근거 인용 |
| **4 — Q&A·최적화** | 12개월+ | 읽기전용 tool-calling Q&A, 모델 라우팅·eval 품질 관리 |

## 7. 주요 리스크 & 완화

| 리스크 | 완화 |
|---|---|
| AI 환각 | 계산-해석 분리 + evidence_ref 강제 + 후처리 검증기(CRO 대조, 재생성) + confidence/caveats |
| Garbage-in | ValidationEngine 치명 오류 시 AI 게이트 차단, 입력 해시 추적 |
| 민감정보 유출 | AI 전송 전 비식별화, RLS+벡터필터 격리, 컬럼 암호화, ZDR/보존정책, 키는 백엔드만 |
| 권한 경계 붕괴 | 필드 레벨 RBAC+마스킹, 가드·RLS 이중화, 전수 감사 |
| 비용·지연 | prompt caching, Batches(50%), 모델 라우팅, effort 조절 |
| 감사·규제 | EvidenceLedger 불변 로그로 전 계보 역추적 |
| 프롬프트 인젝션 | AI 입력을 구조화 CRO로 한정, 읽기전용 도구, 출력 스키마 검증 |
| 조직 수용성 | Human-in-the-loop 강제, 근거 하이라이트 UI, 부서 챔피언 |
